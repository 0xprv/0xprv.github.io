# LLM Calling Agent using RAG + PJSUA + TTS

**Published:** February 1, 2025  
**Status:** 🔄 Currently in Development  
**Project Type:** Startup / Freelance Work

---

## Overview

This project demonstrates the automation of calling people using a combination of Large Language Models (LLM), a SIP server (PJSUA), LangGraph, and Text-to-Speech (TTS) technology. The system enables AI agents to make and handle voice calls autonomously, conducting conversations with contextual awareness and human-like interaction.

**Current Focus:** Optimizing the server to stream voice with lower latency and to handle a variety of prompts using the LLM with improved accuracy and performance.

## Architecture Overview

The LLM Calling Agent operates through an integrated workflow combining multiple technologies:

1. **Voice Input** → SIP PJSUA receives incoming call
2. **Speech-to-Text** → OpenAI Whisper converts audio to text
3. **LLM Processing** → LLaMA 8.1B generates contextual response
4. **Text-to-Speech** → Coqui TTS converts response to audio
5. **Voice Output** → Stream audio back to caller via PJSUA

### Agent Configuration Details

Each agent has specific configuration options that determine its behavior:

#### 1. **Speech to Text**
Toggle to enable/disable conversion of incoming voice streams into text, which is then passed to the LLM.

#### 2. **Gender**
Defines the agent's persona (e.g., male or female), allowing the LLM to adjust tone, language patterns, and communication style accordingly.

#### 3. **Language**
Specifies the language the LLM should use for generating responses and the TTS engine should use for audio output.

#### 4. **Keywords**
An array of important trigger words that initiate human transfer. If any of these keywords appear in the user's query, control is immediately transferred from the AI model to a human operator.

**Example:** A healthcare agent with keyword "cancer" would route urgent health concerns to a real medical professional.

#### 5. **Security Questions**
A list of sensitive prompts that trigger security alerts. If a user query matches these patterns (e.g., requesting credentials, accessing secure information), a warning is sent to the admin, and the conversation is halted to prevent security risks.

#### 6. **Database Connection**
Configuration parameters to connect the agent to a specific database. This enables the agent to:
- Fetch table data and use it as contextual knowledge
- Update records based on conversation outcomes
- Provide intelligent, domain-specific responses

Useful for specialized agents like doctors accessing patient records or lawyers accessing case files.

#### 7. **Files Location**
Specifies a folder path where the agent can access relevant files. The system:
- Extracts and processes file contents using recursive splitting
- Converts files into document embeddings
- Integrates information into system prompts
- Provides enriched context for conversations

**Example:** A restaurant agent could access menus, pricing, and availability schedules from local files.

## Workflow (How It Works)

### Call Flow Process

**1. Call Initiation**
The user initiates a call to the AI agent, which is deployed and running on a Linux Ubuntu server. The agent launches using the configuration parameters set through the UI during setup.

**2. Call Acceptance**
Once the AI agent receives the call, it accepts it using a purchased SIP credential, enabling VoIP functionality through PJSUA.

**3. Audio Capture**
The script begins receiving the user's voice stream via PJSUA2 and simultaneously converts the audio to text using OpenAI Whisper.
- **Processing Time:** ~2–3 seconds per 15 seconds of continuous speech

**4. LLM Processing**
The transcribed text is passed to the LLM (LLaMA 8.1B), which:
- Generates contextually appropriate responses
- References connected databases and documents
- Checks for keyword triggers (requires human transfer)
- Performs security checks on sensitive queries
- Can send emails and update databases as needed

**5. Text-to-Speech Conversion**
The response text is converted into a .wav audio file in the selected language using Coqui TTS.

**6. Audio Streaming**
The audio file is streamed byte-by-byte back to the caller in real-time. This process continues until the LangGraph state machine terminates.

**7. Post-Call Actions**
After the call ends:
- Conversation logs are saved
- Database updates are applied
- Email notifications are sent if configured
- Call metrics are recorded

## Example Conversation

Here's a realistic example of a healthcare agent in action:

```
AGENT: Good morning, how can I help you?

CLIENT: Hi, I've been feeling unwell since yesterday.

AGENT: What symptoms do you have?

CLIENT: Headache, stomachache, and a fever.

AGENT: Do you know your temperature?

CLIENT: Around 40°C.

AGENT: In that case, I'll transfer you to our emergency team.
(OR, if not urgent:)

AGENT: Who is your doctor?

CLIENT: Dr. Pepper.

AGENT: When would you like to come in?

CLIENT: As soon as possible—today or tomorrow.

AGENT: I have an opening today at 4 PM. Does that work?

CLIENT: Perfect.

AGENT: May I have your full name and birthdate?

CLIENT: Frank Muller, born October 8, 1980.

AGENT: Could you spell your last name?

CLIENT: M-U-L-L-E-R.

AGENT: Thanks. Can you confirm your address and phone number?

CLIENT: 10 Washington Street, NYC. Phone: 01 233 44 55.

AGENT: Got it. And your insurance is ASSURA?

CLIENT: Yes.

AGENT: Great. Your appointment with Dr. Pepper is confirmed for today at 4 PM. 
You'll get an SMS confirmation. Have a nice day!
```

**Post-Call Actions:**
- Appointment details are saved to the database
- Confirmation email is sent to the clinic
- SMS reminder is scheduled for the patient
- Call duration is logged for invoicing

## Technical Challenges & Solutions

### Challenge 1: Low-Latency Voice Streaming
**Problem:** Real-time audio streaming with <200ms latency
**Solution:** Implement buffered streaming with PJSUA2, optimize Whisper inference

### Challenge 2: LLM Response Quality
**Problem:** Generating contextually accurate responses across domains
**Solution:** Fine-tune prompts per agent type, implement RAG with domain-specific documents

### Challenge 3: Natural Voice Output
**Problem:** TTS audio quality and prosody
**Solution:** Use Coqui TTS with multi-speaker models, add post-processing effects

### Challenge 4: Scalability
**Problem:** Handling multiple concurrent calls
**Solution:** Kubernetes orchestration, GPU time-sharing, async processing
